# 页面结构与交互设计

## 1. UI 总体原则

Taskora 的 UI 应该像一个安静的工作工具，而不是营销页面或监控后台。

设计关键词：

- 轻量。
- 可扫读。
- 桌面友好。
- 用户可控。
- 不制造压力。
- 不使用“监控”语气。

视觉建议：

- 用浅色中性背景作为默认主题，配少量状态色。
- 任务卡片使用低圆角、轻阴影、紧凑排版。
- 便签可以有淡色背景，但不要做成强烈便利贴玩具风。
- 状态色建议：待办灰、进行中蓝绿、暂停琥珀、完成绿色、逾期红。
- 统计界面以表格、时间线和紧凑图表为主，避免夸张大卡片。

## 2. 信息架构

主要入口：

- 桌面任务便签。
- 系统托盘菜单。
- 主窗口。

主窗口页面：

- Today 今日。
- Tasks 任务。
- Timeline 时间线。
- Knowledge 知识库。

MVP 可以先实现：

- Today。

## 3. 桌面任务便签

### 3.1 便签默认布局

内容：

- 标题。
- 状态。
- 今日用时。
- 累计用时。
- 今日进展数量。
- 截止时间。
- 小型操作按钮：开始/暂停、完成、更多。

示例：

```text
修复登录 Bug
进行中 · 今日 1h 25m · 累计 3h 10m
今日记录 3 条 · 截止 今天 18:00
```

### 3.2 便签状态

待办：

- 显示“开始”按钮。
- 时间弱显示。

进行中：

- 显示运行计时。
- 边框或左侧状态条高亮。

暂停：

- 显示“继续”按钮。
- 说明最近暂停时间。

已完成：

- 显示完成时间。
- 提示可总结/归档。

### 3.3 便签行为

- 拖动：移动桌面位置，停止拖动后保存。
- 双击：打开任务详情。
- 右键：打开上下文菜单。
- 折叠：只显示标题和状态。
- 关闭：隐藏便签，不删除任务。
- 删除：必须在任务详情或菜单中二次确认。

## 4. 右键菜单

菜单项：

```text
开始专注
暂停任务
添加进展
标记完成
延后到明天
关联当前软件
查看任务时间线
生成任务总结
归档到知识库
```

交互规则：

- 根据任务状态启用或隐藏不适用项。
- “标记完成”后弹出完成确认，可选择立即生成任务总结。
- “关联当前软件”把当前前台应用加入任务关联规则，MVP 先只保存人工关联，不做自动推断。

## 5. 添加进展弹窗

字段：

- 完成了什么？
- 遇到了什么问题？
- 下一步做什么？

自动上下文区域：

```text
时间：14:32
当前软件：Visual Studio Code
当前窗口：login.tsx
当前任务：修复登录 Bug
```

隐私命中时显示：

```text
当前软件：隐私应用
当前窗口：已隐藏
```

按钮：

- 保存。
- 保存并暂停。
- 取消。

## 6. Today 今日视图

目的：

让用户快速理解今天的工作，而不是制造复杂报表。

模块：

- 顶部状态：记录中/已暂停、今日活跃时长、空闲时长。
- 任务投入：任务列表、今日用时、进展数、状态。
- 应用投入：应用名、用时、切换次数。
- 时间线：按时间段显示应用和任务。
- 今日进展：按时间倒序。
- AI 总结按钮。

Today 的 AI 总结流程：

1. 点击“生成今日总结”。
2. 打开数据范围确认页。
3. 用户选择是否包含应用标题、private 数据默认不可选。
4. 选择 AI provider。
5. 生成 Markdown 草稿。
6. 用户编辑。
7. 保存到 AI Summaries。
8. 可归档到 Knowledge。

## 8. Timeline 时间线视图

MVP 可以简单实现：

- 按日期显示时间段。
- 每段包含时间、应用、窗口标题、任务、是否 idle/private。
- 支持按应用、任务、private、idle 筛选。

文案：

- 使用“工作时间线”。
- 不使用“监控日志”。

- Progress Note。
- Project Note。

## 10. Settings 设置

设置分组：

- General：开机启动、语言、主题。
- Recording：采样间隔、空闲阈值、记录开关。
- Privacy：隐私软件、窗口标题关键词、AI 数据确认、删除数据。
- AI：provider、endpoint、model、API key、本地模型。
- Knowledge Base：目录、Markdown 导出策略。

隐私设置应放在一级页面中，不能藏太深。

## 11. 首次启动流程

1. 欢迎：Taskora 是本地优先的个人工作记忆工具。
2. 隐私说明：默认只记录应用和窗口标题，不记录键盘、截图、剪贴板。
3. 设置隐私软件黑名单：微信、QQ、Telegram、密码管理器、银行/支付软件。
4. 选择 AI provider：跳过、OpenAI-compatible、DeepSeek、Ollama。
5. 创建第一个任务。

## 12. 微文案建议

推荐：

- “暂停工作记录”
- “今天的工作记忆”
- “生成复盘草稿”
- “确认发送给 AI 的内容”
- “隐藏此应用的窗口标题”
- “删除这一天的记录”

避免：

- “开始监控”
- “审计用户行为”
- “捕获屏幕内容”
- “监听输入”
- “员工效率评估”
