# MVP 功能清单

## MVP 目标

MVP 只验证一个闭环：

用户把任务挂在桌面，围绕任务工作，Taskora 记录轻量上下文，用户补充关键进展，任务完成后生成总结并沉淀到本地知识库。

## 必须第一版就做

### 1. 任务便签

- 创建任务。
- 编辑任务标题、描述、截止时间、项目、标签。
- 桌面显示任务便签。
- 便签位置保存和恢复。
- 任务状态：待办、进行中、暂停、已完成。
- 今日用时和累计用时显示。

验收标准：

- 重启应用后，便签仍在上次位置。
- 任务状态和用时显示准确。
- 至少支持 10 个同时存在的便签，不明显卡顿。

### 2. 任务计时

- 开始任务。
- 暂停任务。
- 完成任务。
- 记录 task session。
- 允许同一时间只运行一个主任务。

验收标准：

- 开始任务会创建会话。
- 暂停或完成会结束会话。
- 非正常退出后，下次启动能把未结束会话标记为 interrupted。

### 3. 右键添加进展

- 表单字段：完成了什么、遇到了什么问题、下一步做什么。
- 自动附带当前时间、当前软件、当前窗口标题、当前任务。

验收标准：

- 进展会出现在任务时间线。
- AI 任务总结能读取这些进展。
- 命中隐私规则时不保存原始窗口标题。

### 4. 前台应用记录

- 采样当前前台窗口。
- 记录进程名、窗口标题、开始结束时间。
- 聚合为 activity span。
- 检测用户空闲。
- 命中隐私软件黑名单时脱敏。

验收标准：

- 不记录键盘内容、截图、剪贴板。
- 每日应用用时统计与实际大致一致。
- 常驻 CPU 低，数据库写入不会频繁抖动。

### 5. 今日统计

- 按任务统计今日用时。
- 按应用统计今日用时。
- 展示时间线。
- 展示进展记录。
- 展示已完成和待继续任务。

验收标准：

- 用户能从今日视图看出一天主要工作。
- private 数据在统计中只显示为“隐私应用”或“已隐藏”。

### 6. AI 今日总结与任务总结

- 支持配置至少一个 OpenAI-compatible endpoint。
- 支持 Ollama/local endpoint 作为可选 provider。
- AI 输入前展示数据范围。
- 输出 Markdown。
- 用户可编辑后保存。

验收标准：

- 今日总结包含已完成、待继续、主要时间投入、打断/切换提示、明日建议。
- 任务总结包含任务过程、关键进展、问题、结果、后续建议。
- AI 不得凭空声称读取了未提供的内容。

### 7. 本地知识库归档

- 完成任务后可归档。
- AI 总结可归档。
- 知识条目写入 SQLite。
- 可导出 Markdown 文件。
- 支持全文搜索。

验收标准：

- 搜索任务名、进展关键词可以找到相关归档。
- Markdown 导出内容可读，包含元数据。

### 8. 基础隐私设置

- 暂停所有记录。
- 隐私软件黑名单。
- 窗口标题关键词黑名单。
- 删除某一天记录。
- 删除某个任务记录。
- AI 发送前确认数据范围。

验收标准：

- 用户随时能看到当前是否正在记录。
- 命中黑名单后不会保存原始标题。
- 删除记录后，今日统计、知识库索引能同步更新。

## 第一版应该砍掉

- 浏览器插件和网站域名识别。
- OCR、截图、录屏。
- 键盘输入和鼠标轨迹。
- 剪贴板记录。
- 自动读取聊天内容。
- 本地向量库强依赖。
- 团队协作和后台管理。
- 跨设备同步。
- 复杂项目管理功能。
- 自动判断“效率高低”的强结论。

## 可选但不阻塞 MVP

- 番茄钟。
- 开机启动。
- 多主题。
- Markdown 批量导出。
- 应用分类。
- 任务自动关联规则。
- 每周总结。

## MVP 用户路径

1. 用户首次启动 Taskora。
2. 看到隐私说明和本地优先承诺。
3. 创建第一个任务“修复登录 Bug”。
4. 任务便签出现在桌面。
5. 用户右键点击“开始专注”。
6. Taskora 开始记录该任务会话，同时轻量记录前台应用。
7. 用户中途右键“添加进展”。
8. 用户暂停或完成任务。
9. 完成后 Taskora 生成任务总结草稿。
10. 用户保存总结并归档到知识库。
11. 晚上用户打开今日复盘。
12. 用户确认 AI 输入范围，生成今日总结。

## MVP Release Definition

可以发布 MVP 的条件：

- 数据可以连续记录一整天，不丢任务会话。
- 隐私黑名单可靠生效。
- AI 总结可以保存和再次查看。
- 数据库结构支持后续扩展，不需要为了周报和知识库问答大改。
- UI 虽然简单，但用户能完成完整闭环。
